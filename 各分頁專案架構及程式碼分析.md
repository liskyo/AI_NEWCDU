## 1. 系統全域功能 (Global Features)

本系統設計核心為提供直觀且強大的操作體驗，並具備全天候無人值守的顯示能力。

### 1.1 導覽列 (Navigation Bar)
-   **位置**: 畫面頂端，各分頁切換的入口。
-   **功能**:
    -   顯示 Guchii Logo 與專案名稱。
    -   提供主要功能分頁連結 (Status, Control, Trends 等)。
    -   **Kiosk 模式切換鈕**: 電腦螢幕圖示 (🖥️)，一鍵進入戰情室模式。

### 1.2 戰情室模式 (Kiosk Mode / War Room Display)
-   **用途**: 適用於大型顯示器或無人監控站，自動輪播關鍵資訊。
-   **觸發方式**: 點擊導覽列圖示或按下 `F11` (瀏覽器全螢幕)。
-   **運作邏輯**:
    -   隱藏導覽列與視窗邊框，最大化可視區域。
    -   **自動輪播 (Auto-Rotation)**: 系統會自動在 **Status (狀態頁)** 與 **Trends (趨勢頁)** 之間切換。
    -   **輪播間隔**: 固定為 **30 秒**。
-   **退出方式**: 按下鍵盤 `ESC` 鍵即可隨時返回標準操作模式。

### 1.3 即時狀態指示
-   **連線狀態**: 畫面右上角或特定區域會顯示與後端/PLC 的連線燈號。
    -   🟢 **ONLINE**: 通訊正常，數據即時更新。
    -   🔴 **OFFLINE / ERROR**: 通訊中斷，顯示快取數據或警示。

## 2. Status: 系統狀態監控

**檔案**: `StatusEngineer.vue`
**路徑**: `/status`

此頁面為 CDU 的數位雙生 (Digital Twin) 面板，將物理世界的感測器數值以圖形化方式呈現。

### 2.1 視覺化系統圖 (System Diagram)
-   **動態元件**:
    -   **水箱 (Tank)**: 根據水位高低動態填充藍色液體動畫。
    -   **管路 (Pipes)**: 當 Pump 運作時，管路會有流動動畫效果。
    -   **溫度計 (Thermometer)**: 顏色隨溫度變化 (藍 -> 綠 -> 紅)。
-   **顯示模式**:
    -   支援淺色 (Light) 與深色科技風 (Dark/Cyberpunk) 兩種底圖切換。

### 2.2 關鍵監控數據 (Read-Only Data Points)
以下數據每 1-5 秒從 PLC 讀取一次：

#### 冷卻液迴路 (Coolant Loop / Secondary Side)
| 參數名稱 | 代號 | 單位 | 描述 |
| :--- | :--- | :--- | :--- |
| **供水溫度** | T1 | °C | 流向伺服器的冷卻液溫度 |
| **回水溫度** | T2 | °C | 從伺服器流回的冷卻液溫度 |
| **供水壓力** | P1 | kPa | Pump 出口壓力 |
| **回水壓力** | P2 | kPa | 伺服器回流壓力 |
| **流量** | F1 | LPM | 冷卻液循環流量 |
| **導電度** | Conductivity | µS/cm | 水質導電度監測 |
| **酸鹼值** | pH | pH | 水質酸鹼度 |

#### 設施水迴路 (Facility Water / Primary Side)
| 參數名稱 | 代號 | 單位 | 描述 |
| :--- | :--- | :--- | :--- |
| **進水溫度** | T4 | °C | 廠務端供給的冰水溫度 |
| **出水溫度** | T5 | °C | 熱交換後排出的溫水溫度 |
| **進水壓力** | P10 | kPa | 廠務端供給壓力 |
| **比例閥** | PV1 | % | 設施水流量控制閥開度 (0-100%) |

#### 其他感測器
-   **環境溫度 (Ambient Temp)**: 機櫃周遭環境溫度。
-   **露點溫度 (Dew Point)**: 計算出的結露風險溫度。
-   **變頻器頻率 (Inverter Freq)**: Pump 1 & 2 的即時運轉頻率 (Hz)。

## 3. Control: 運轉模式控制

**檔案**: `Control.vue`
**路徑**: `/control`

此頁面為 CDU 的大腦，允許授權人員接管系統控制權。提供三種主要操作模式。

### 3.1 模式選擇 (Mode Selection)
1.  **Auto (自動模式)**: 系統依據設定的目標值，自動調節 Pump 轉速與閥門開度 (PID 控制)。
2.  **Manual (手動模式)**: 使用者直接控制硬體輸出，不經過 PID 運算。
3.  **Stop (停機模式)**: 安全停止所有 Pump 與閥門動作。
4.  **Engineer (工程模式)**: (僅顯示提示，需切換至 Engineer 分頁操作)。

### 3.2 可調整參數 (Adjustable Parameters)

#### 🔵 Auto Mode 設定
在此模式下，PID 控制器會嘗試將感測器數值維持在以下設定點：
-   **目標冷卻液溫度 (Target Coolant Temp)**:
    -   **變數**: `inputs.temp`
    -   **單位**: °C
    -   **說明**: 期望 T1 供水溫度的恆定值。
-   **目標冷卻液壓力 (Target Coolant Pressure)**:
    -   **變數**: `inputs.pressure`
    -   **單位**: kPa
    -   **說明**: 期望 P1 供水壓力的恆定值 (定壓控制)。

#### 🟡 Manual Mode 設定
強制輸出固定訊號，適用於測試或 PID 失效時的緊急應變：
-   **Pump 1 轉速**:
    -   **變數**: `inputs.pump1`
    -   **範圍**: 0 - 100 %
-   **Pump 2 轉速**:
    -   **變數**: `inputs.pump2`
    -   **範圍**: 0 - 100 %
-   **設施水比例閥 (Water PV) 開度**:
    -   **變數**: `inputs.water_pv`
    -   **範圍**: 0 - 100 % (0=全關, 100=全開)

#### 🔴 Stop Mode 維護開關
在系統停機狀態下，可單獨對變頻器進行通電測試或維護：
-   **Inverter 1 Maintenance Switch**: 開/關
-   **Inverter 2 Maintenance Switch**: 開/關

### 3.3 系統操作功能
-   **EXECUTE (執行)**: 將上述設定寫入 PLC，立即生效。
-   **ERROR RESET (錯誤復歸)**: 清除 PLC 的暫時性錯誤旗標 (如過溫警報後的手動復歸)。
-   **FIRMWARE UPDATE (韌體更新)**:
    -   **支援格式**: `.zip` 壓縮檔。
    -   **流程**: 選擇檔案 -> 上傳 -> 後端解壓並執行更新腳本 -> 自動重啟服務。
-   **REBOOT SYSTEM**: 遠端重啟 CDU 控制器 (Linux/Windows)。

## 4. Trends: 歷史趨勢分析

**檔案**: `Trends.vue`
**路徑**: `/trends`

提供數據可視化，協助工程師分析系統穩定性與 PID 調校成果。

### 4.1 圖表功能
-   **SVG 繪圖引擎**: 使用原生 SVG Path 繪製，效能優於 Canvas，支援無限縮放不失真。

-   **資料來源切換**:
    -   **LIVE**: 顯示真實感測器數據歷史。

### 4.2 互動控制

-   **Legend Toggle**: 點擊圖例可顯示/隱藏特定曲線 (如只看 T1, T2)。

## 5. Maintenance: 維護與壽命管理

**檔案**: `Maintenance.vue`
**路徑**: `/maintenance`

預防性維護系統，追蹤關鍵耗材的使用時數，避免意外停機。

### 5.1 監控項目 (Component Lifecycle)
系統追蹤以下零件的「已運轉時數」與「預計最大壽命」：

1.  **Main Pump 1 (主幫浦 1)**
    -   最大壽命: 10,000 小時
2.  **Main Pump 2 (主幫浦 2)**
    -   最大壽命: 10,000 小時
3.  **Fine Filter Cartridge (精密濾心)**
    -   最大壽命: 1,000 小時 (需頻繁更換)
4.  **Coolant Fluid (冷卻液)**
    -   最大壽命: 8,000 小時 (化學性質衰退週期)
5.  **Solenoid Valve Set (電磁閥組)**
    -   最大壽命: 20,000 小時

### 5.2 健康度邏輯 (Health Logic)
-   $$健康度 (\%) = \max(0, 100 - (\frac{目前時數}{最大壽命} \times 100))$$
-   **狀態燈號**:
    -   🟢 **> 50%**: 良好 (Good)
    -   🟡 **20% - 50%**: 警告 (Warning)
    -   🔴 **< 20%**: 危險 (Critical) - 建議立即更換

### 5.3 維護操作
-   **Record Maintenance (紀錄保養)**: 更換零件後，點擊此按鈕可將計數器歸零 (Reset to 0 hours)。
-   **Service History**: 顯示過往的維修紀錄列表 (日期、項目、備註)。

## 6. Backup: 系統備份與還原

**檔案**: `Backup.vue`
**路徑**: `/backup`

確保系統設定的安全性，支援災難復原。

### 6.1 Backup (匯出設定)
-   **功能**: 將所有 PLC 參數、閾值、PID 設定打包為 JSON 檔案。
-   **內容包含**:
    -   Sensor Offsets
    -   Alarm Thresholds
    -   PID Kp, Ki, Kd
    -   Network Config

### 6.2 Restore (還原設定)
-   **功能**: 上傳先前備份的 JSON 檔案。
-   **流程**: 檔案解析 -> 數值範圍檢查 -> 逐一寫入 PLC -> 驗證寫入結果。

## 7. Network: 網路通訊配置

**檔案**: `Network.vue`
**路徑**: `/network`

管理 CDU 的對外通訊能力，包含 SCADA 連線與 Modbus 整合。

### 7.1 Modbus Master 設定
-   **Modbus IP Address**: 設定 CDU 作為 Master 去讀取底層 PLC 時的目標 IP。
    -   預設: `192.168.1.50` (依現場 PLC 設定調整)

### 7.2 介面配置 (Interface Config)
支援多組網卡 (ETH1, ETH2, USB-LAN) 的獨立設定。

#### IPv4 設定
-   **DHCP**: 開啟/關閉 (自動取得 IP)。
-   **Static IP**: 手動指定 IP 位址。
-   **Subnet Mask**: 子網路遮罩。
-   **Gateway**: 預設通訊閘。
-   **DNS**: 主/次 DNS 伺服器 (支援 Auto/Manual)。

#### IPv6 設定
-   支援完整的 IPv6 Address, Prefix, Gateway 設定，符合新一代資料中心規範。

### 7.3 狀態圖示
-   顯示各介面的實體連結狀態 (Link Up/Down)。

## 9. Engineer Mode: 工程師進階設定

**檔案**: `EngineerMode.vue`
**路徑**: `/engineer-mode`

**⚠️ 警告**: 此區域參數直接影響控制邏輯核心，僅限授權工程師進入。

詳細參數列表如下：

### 9.1 Sensor Adjustment (感測器校正)
針對 9 組主要感測器 (T1, T2, P1, P2 等)，可設定：
-   **Factor (增益)**: 線性校正斜率 (預設 1.0)。
-   **Offset (偏移)**: 線性校正截距 (+/- 數值)。
-   *用途: 當感測器老化或更換新批次時，進行軟體歸零校正。*

### 9.2 Threshold Setting (閾值設定)
設定觸發警報的邊界值。每顆感測器皆有兩階段閾值：
1.  **Warn Limit (警告值)**: 觸發黃色警示，系統維持運轉。
2.  **Alert Limit (危險值)**: 觸發紅色警報，系統可能執行保護停機。

### 9.3 PID Control Params (PID 參數調校)
獨立調整 **壓力 (Pressure)** 與 **溫度 (Temperature)** 兩組控制迴路。
-   **Sample Time**: 取樣時間 (ms)。
-   **KP (Proportional)**: 比例參數，決定反應速度。
-   **Integral Time / KI**: 積分參數，消除穩態誤差。
-   **Derivative (KD)**: 微分參數，抑制過充 (Overshoot)。
-   **Derivative Time**: 微分時間常數。

### 9.4 Auto Mode Redundancy (自動模式備援)
設定當自動控制失效時的保底 (Failsafe) 行為：
-   **Water Valve (%)**: 失效時閥門維持的開度。
-   **Pump Speed (%)**: 失效時 Pump 維持的轉速。
-   **Min Opening (Dew Point)**: 避免結露的最小閥門開度限制。

### 9.5 Valve Control Logic (閥門邏輯)
設定閥門強制關閉的保護條件：
-   **Ambient T(a) < X**: 當環境溫度低於 X 度時強關閥門。
-   **Coolant T(1) < Y**: 當冷卻液溫度低於 Y 度時強關閥門。
-   *用途: 防止過度冷卻導致伺服器結露。*

### 9.6 Version Control (版本控制)
切換不同的控制策略演算法 (A/B Testing)：
-   **Function Switch**: Legacy / Modern
-   **Flow Switch**: Old / New
-   **Median Filter**: 開啟/關閉中值濾波器 (抗雜訊)。

## 10. Settings: 系統一般設定

**檔案**: `Settings.vue`
**路徑**: `/settings`

### 10.1 Unit Standard (單位設定)
-   **Metric (公制)**: °C, kPa, LPM (預設)。
-   **Imperial (英制)**: °F, psi, GPM。

### 10.2 System Time (時間設定)
-   顯示目前控制器時間。
-   **Manual Set**: 手動輸入時間。
-   **Synchronize**: 一鍵同步瀏覽器時間到控制器。

### 10.3 Data Logging (資料記錄)
-   **Sampling Interval**: 設定後端記錄歷史數據的頻率。
    -   **範圍**: 2秒 - 30秒。
    -   *注意: 頻率過高可能佔用磁碟空間，過低可能遺漏瞬間異常。*

### 10.4 SNMP Configuration
設定北向介接 (Northbound Interface) 資訊，讓 DCIM 系統監控 CDU。
-   **Trap Server IP**: 接收主動告警的伺服器 IP。
-   **Read Community String**: SNMP 讀取權限字串 (預設 public)。

### 10.5 Security (安全性)
-   **Change Password**: 修改目前登入使用者的密碼。
-   **Reset Default**: 恢復原廠預設密碼 (需實體存取權限)。

## 11. 技術架構附錄 (Technical Appendix)

### 11.1 軟體堆疊詳細規格
| Layer | Technology | Details |
| :--- | :--- | :--- |
| **Frontend** | Vue 3 + Vite | 使用 Composition API (`<script setup>`) 開發，確保代碼模組化。 |
| **UI Framework** | Tailwind CSS | 採用 Utility-first 設計，快速建構 RWD 與深色模式介面。 |
| **Icons** | Heroicons | 使用標準 SVG 圖示庫，保持視覺一致性 (Solid/Outline)。 |
| **Backend** | Python Flask | 輕量級 Web Server，負責 API 路由與靜態檔服務。 |
| **Protocol** | Pymodbus TCP | 透過 Modbus TCP 協定與 PLC (Port 502) 通訊。 |
| **Database** | SQLite + JSON | 日誌使用 SQLite 儲存，設定檔使用 JSON 格式序列化。 |
| **Deployment** | Nginx + Gunicorn | 生產環境使用 Nginx 作為 Reverse Proxy，Gunicorn 作為 WSGI Server。 |

### 11.2 API 路由表 (API Endpoints)
前端與後端溝通的主要 API 列表：

**資料讀取 (GET)**
-   `/get_data_control`: 獲取控制頁面所需的完整狀態 (包含 Mode, Pump Speed, Valve)。
-   `/get_data_engineerMode`: 獲取工程模式所需的原始感測器數據與閾值。
-   `/get_data_systemset`: 獲取系統設定 (單位、Log 頻率)。
-   `/get_logs`: 分頁獲取系統日誌。
-   `/get_network_info`: 獲取網卡資訊。

**指令寫入 (POST)**
-   `/set_operation_mode`: 切換運轉模式 (Auto/Manual/Stop) 並寫入參數。
-   `/writeSensorAdjust`: 寫入感測器校正值 (Factor/Offset)。
-   `/thrshd_set`: 寫入警報閾值。
-   `/store_pid`: 寫入 PID 控制參數。
-   `/update_password`: 修改使用者密碼。
-   `/api/v1/1.3MW/upload_zip`: 上傳並執行韌體更新。

### 11.3 資料流向 (Data Flow Architecture)

1.  **Poll Loop (Backend)**: 
    -   Python 後臺執行緒 (`threading`) 每秒輪詢 PLC 暫存器。
    -   解析 Binary Payload 並轉換為浮點數。
    -   將數據快取於 Python 全域變數或記憶體資料庫 (`Redis`/`Dict`)。
2.  **API Request (Frontend)**: 
    -   Vue 前端透過 `axios` 每 1-5 秒 (可調) 呼叫 `/get_data_control`。
    -   前端取得 JSON 格式的最新快取數據，避免直接對 PLC 發起高頻請求。
3.  **Command (User)**: 
    -   使用者操作介面 -> 呼叫 POST API。
    -   API 接收請求 -> 驗證參數範圍 -> 寫入 PLC 暫存器 (Coil/Register)。
    -   寫入成功後，回傳 Success 訊號給前端。
4.  **Notification**: 
    -   後端監控線程偵測到異常 (如 T1 > Alert Limit)。
    -   觸發 SNMP Trap 推播至 DCIM。
    -   寫入 SQLite 日誌資料庫。
    -   前端輪詢時發現異常旗標，顯示橫幅警報。

## 12. 常見問題排除 (Troubleshooting Guide)

本節提供常見問題的故障排除步驟，供維護人員參考。

### 12.1 系統連線問題
**徵狀**: 畫面右上角顯示🔴 **OFFLINE**，數據停止更新。
**可能原因**:
1.  後端服務 (Flask) 已停止。
2.  PLC 斷線或 IP 變更。
3.  網路線脫落。
**排除步驟**:
1.  檢查主機網路線是否連接妥當。
2.  嘗試 Ping PLC IP (預設 `192.168.1.50`)。
3.  若無法 Ping 通，檢查 Network 分頁的本機 IP 設定是否在同一網段。
4.  若 Ping 通但仍 OFFLINE，嘗試重啟後端服務或整機重啟 (Reboot)。

### 12.2 感測器數值異常
**徵狀**: 某個溫度顯示 `-999` 或與實際環境差異過大。
**可能原因**:
1.  感測器斷線 (Broken Wire)。
2.  校正參數 (Offset/Factor) 設定錯誤。
3.  感測器損壞。
**排除步驟**:
1.  進入 **Engineer Mode** > **Sensor Adjustment**。
2.  檢查該感測器的 Factor 是否為 `1.0`，Offset 是否為 `0.0`。
3.  若參數正常，請檢查硬體接線端子是否鬆脫。
4.  若接線正常，可能需更換感測器。

### 12.3 無法切換至 Auto 模式
**徵狀**: 點擊 `Auto` 按鈕並執行 `Execute` 後，模式跳回 `Stop` 或 `Manual`。
**可能原因**:
1.  系統存在未復歸的 **Critical Alarm** (如漏水、過溫)。
2.  PLC 邏輯鎖定 (Interlock)。
**排除步驟**:
1.  觀察導覽列是否有紅色警報橫幅。
2.  進入 **Status** 分頁確認具體警報內容。
3.  排除故障源頭 (如修復漏水)。
4.  進入 **Control** 分頁，點擊 **ERROR RESET** 按鈕。
5.  再次嘗試切換至 Auto 模式。

### 12.4 流量計讀數為 0
**徵狀**: Pump 已運轉 (轉速 > 0%)，但流量計 (F1/F2) 顯示 0 LPM。
**可能原因**:
1.  管路內有空氣 (氣鎖 Air Lock)。
2.  閥門未開啟。
3.  流量計訊號線故障。
**排除步驟**:
1.  檢查管路閥門是否全開。
2.  執行排氣程序 (Air Purge)。
3.  進入 **Engineer Mode** 檢查 **Flow Switch** 版本設定是否正確 (Old/New)。

### 12.5 設定無法儲存
**徵狀**: 修改參數後點擊 Save，重新整理後數值回復原狀。
**可能原因**:
1.  PLC 處於唯讀模式或寫入保護。
2.  輸入數值超出 PLC 允許範圍。
**排除步驟**:
1.  確認輸入數值是否在合理範圍內 (參考各分頁說明)。
2.  檢查 Modbus 連線狀態。
3.  查看瀏覽器 Console (F12) 是否有 API Error (400/500)。
