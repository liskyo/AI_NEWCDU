# CDU 專案架構及程式碼分析 (各分頁詳解)

**檔案名稱**: 各分頁專案架構及程式碼分析.md
**專案描述**: Guchii CDU Control (冷卻分配單元控制系統) Web 介面
**技術堆疊**: Vue 3 (Composition API), Vite, Tailwind CSS, Axios, Heroicons

---

## 1. 專案全域架構 (Global Architecture)

### 1.1 App.vue (主應用程式入口)
- **職責**: 定義全域佈局 (Layout)，包含導覽列 (Navbar) 與主內容區 (Router View)。
- **核心功能**:
    - **RWD 導覽列**: 包含所有分頁的連結與 Icon。
    - **Kiosk Mode (戰情室模式)**:
        - 透過 `isKioskMode` 狀態控制，切換為全螢幕並隱藏導覽列。
        - 實作 `requestFullscreen` API 呼叫。
        - **自動輪播機制**: 在 `Status` 與 `Trends` 頁面間每 30 秒自動切換。
        - **退出機制**: 監聽 `keydown` (ESC) 與 `fullscreenchange` 事件以恢復正常模式。
    - **轉場效果**: 使用 `<transition>` 標籤實現頁面切換的淡入淡出效果 (`fade-enter-active`, `fade-leave-active`)。

### 1.2 Router (router/index.js)
- **職責**: 定義前端路由規則。
- **路徑映射**:
    - `/status` -> `StatusEngineer.vue` (預設首頁)
    - `/control` -> `Control.vue`
    - `/trends` -> `Trends.vue`
    - `/maintenance` -> `Maintenance.vue`
    - `/backup` -> `Backup.vue`
    - `/network` -> `Network.vue`
    - `/logs` -> `Logs.vue`
    - `/engineer-mode` -> `EngineerMode.vue`
    - `/settings` -> `Settings.vue`

## 2. 各分頁詳細分析 (Page Analysis)

### 2.1 Status (StatusEngineer.vue) - 系統狀態首頁
- **功能**: 展示 CDU 即時運轉狀態、系統示意圖 (System Diagram) 與關鍵感測器數值。
- **組件結構**:
    - 引入 `SystemDiagram.vue` (淺色模式) 與 `SystemDiagram_Dark.vue` (深色/科技感模式)。
- **資料流**:
    - **API**: GET `/api/status` (獲取所有感測器數據)。
    - **Test Mode**: 內建 `generateMockData()` 函式，可切換至測試模式生成假資料以驗證 UI。
    - **定時刷新**: 使用 `setInterval` 定期更新數據。
- **UI 特色**:
    - **視覺化**: 大量使用 SVG 動態綁定數據，如水箱水位、溫度計顏色變化。
    - **警報顯示**: 顯示連線狀態與異常訊息橫幅。

### 2.2 Control (Control.vue) - 控制中心
- **功能**: 切換系統運作模式 (Auto, Manual, Stop, Engineer) 並設定對應參數。
- **核心邏輯**:
    - **模式切換**: 透過 Tabs UI 切換 `selectedMode`。
    - **Auto Mode**: 設定目標溫度 (`temp`) 與壓力 (`pressure`)。
    - **Manual Mode**: 手動設定 Pump 轉速 (%) 與 比例閥 (PV) 開度。
    - **Stop Mode**: 系統停機與變頻器保養開關控制。
- **資料流**:
    - **讀取**: GET `/get_data_control`。
    - **寫入**: POST `/set_operation_mode` (包含模式與所有參數設定)。
    - **韌體更新**: 實作檔案上傳 (`input type="file"`) POST `/api/v1/1.3MW/upload_zip`。

### 2.3 Trends (Trends.vue) - 趨勢圖表
- **功能**: 繪製關鍵數據 (溫度、壓力、流量) 的歷史趨勢曲線。
- **核心邏輯**:
    - **SVG 繪圖**: 手動計算 SVG Path (`d` 屬性)，不依賴外部圖表庫，效能最佳化。
    - **Live/Sim 切換**:
        - **LIVE DATA**: 呼叫 `/get_data_engineerMode` 獲取真實數據。
        - **TEST SIM**: 本地生成正弦波 (Sine Wave) 模擬數據，用於展示與測試 UI 反應。
    - **互動功能**: 支援暫停/繼續 (`isPaused`)、切換顯示指標 (Metric)。
- **資料流**:
    - 定時 (1秒) 更新歷史陣列 `historyData` (FIFO Queue)。

### 2.4 Maintenance (Maintenance.vue) - 保養維護
- **功能**: 追蹤主要耗材 (Pump, Filter, Fluid) 的運轉時數與健康度。
- **核心邏輯**:
    - **健康度計算**: `getHealthPercent` 函式，根據運轉時數與最大壽命計算百分比。
    - **視覺化**: 使用進度條與顏色編碼 (綠/黃/紅) 顯示狀態。
    - **重置功能**: `resetPart` 用於保養後歸零計數 (目前僅前端模擬)。
- **資料流**:
    - 目前使用前端 `ref` 定義的 Mock Data (`parts`, `logs`)，尚未串接後端 API。

### 2.5 Backup (Backup.vue) - 備份與還原
- **功能**: 匯出系統設定檔 (JSON) 與還原設定。
- **核心邏輯**:
    - **匯出模擬**: 模擬打包過程 (進度條動畫)，最後動態生成 Blob 下載 `.json` 檔案。
    - **匯入模擬**: 觸發檔案選擇器，模擬上傳與完整性檢查流程。
- **UI 特色**:
    - 大區塊卡片設計，強調備份與還原兩個主要動作。
    - 加入了擬真的處理中 (Processing) 遮罩層與進度條動畫。

### 2.6 Network (Network.vue) - 網路設定
- **功能**: 設定實體網路介面 (Ethernet/USB) 的 IPv4/IPv6 參數與 Modbus IP。
- **核心邏輯**:
    - **Modbus 設定**: 獨立區塊設定 Modbus Master IP。
    - **介面管理**: 列表顯示多個網卡，支援展開/收合詳細資訊。
    - **IP 設定**: 支援 DHCP/Static 切換，欄位包含 IP, Mask, Gateway, DNS。
- **資料流**:
    - **讀取**: `fetchNetworkInfo` (目前由 Mock Data `mockNetworkData` 驅動，因 API 可能未就緒)。
    - **API 預留**: 程式碼中保留了 `axios.get('/get_network_info')` 等路徑。
    - **狀態顯示**: 判斷連線狀態並顯示 `ONLINE`/`OFFLINE` 標籤。

### 2.7 Logs (Logs.vue) - 系統日誌
- **功能**: 表格化顯示系統事件、警報與操作紀錄。
- **核心邏輯**:
    - **分頁 (Pagination)**: 實作前端分頁功能 (`currentPage`, `itemsPerPage`)。
    - **排序**: 支援依日期、等級、訊息內容排序。
    - **自動刷新**: 支援背景自動刷新 (`setInterval`)，並優化了 Loading 狀態 (避免背景刷新時干擾使用者)。
    - **匯出**: CSV 匯出功能。
- **資料流**:
    - **API**: GET `/get_logs`。

### 2.8 Engineer Mode (EngineerMode.vue) - 工程模式
- **功能**: 進階參數設定，包含感測器校正、PID 參數、閥件邏輯、韌體資訊。
- **結構**: 使用手風琴 (Accordion) 收合選單整理大量參數：
    1.  **Sensor Adjustment**: 設定 Offset 與 Factor (POST `/writeSensorAdjust`)。
    2.  **Threshold Setting**: 設定各感測器的警告與危險閾值 (POST `/thrshd_set`)。
    3.  **PID Setting**: 獨立設定溫度與壓力的 PID 參數 `Kp`, `Ki`, `Kd` (POST `/store_pid`)。
    4.  **FW Setting**: 檢視與更新序號、型號 (POST `/write_version`)，查看各子系統版本。
    5.  **Switch Version**: 切換新舊版控制邏輯 (POST `/version_switch`)。
    6.  **Misc**: 閥件關閉條件、指示燈逾時設定。

### 2.9 Settings (Settings.vue) - 系統設定
- **功能**: 一般使用者設定。
- **包含項目**:
    - **單位切換**: 公制/英制 (Metric/Imperial)。
    - **時間設定**: 設定系統時間與時區 (POST `/set_time`)。
    - **密碼更改**: 更新登入密碼 (POST `/update_password`)。
    - **SNMP**: 設定 Trap server IP 與 Community string (POST `/store_snmp_setting`)。
    - **Data Log**: 設定資料取樣頻率 (Sampling Rate)。

## 3. 共通設計模式

### 3.1 HTTP 請求
- 全面使用 `axios` 進行後端通訊。
- 採用 `async/await` 語法處理非同步請求。
- 錯誤處理統一使用 `try/catch` 區塊，並透過 `alert` 或 `console.error` 回饋。

### 3.2 樣式系統
- **Tailwind CSS**: 負責所有樣式，大量使用 `bg-gray-900`, `text-cyan-400` 等類別營造 Cyberpunk 風格。
- **動畫**: 使用 CSS Keyframes (`scanline`, `shimmer`, `pulse`) 增加科技感互動。

### 3.3 圖示系統
- **Heroicons**: 使用 `@heroicons/vue/24/solid` 或 `outline` 套件，統一 UI 視覺語言。
